Final Team Retrospective

Reflection on what we did well as a team

  1. Communication via WhatsApp was our primary mode of both keeping each other up to date on developments as well as raising issues. This generally worked well and was an easier way to track project developments than Github.
  2. Using Zenhub to track user stories also proved effective. The team was able to implement user stories following acceptance criteria and the definition of done (DOD).

What we learned towards the end that could have helped us

  1. We encountered an issue at the end - bots no longer coming through to the site thus preventing realizing results on our A/B test of “donate” vs. “support” language. We brought everyone together into one thread eventually and, with Kyle’s help, identified the issue. We might have gotten here sooner had we had a more vertically aligned process in which one person owned one task; however, this also would require everyone knowing how to do all components (e.g., code, analytics, manage, etc.). To be more efficient in our current process, we might have created a more standardized channel through which to raise issues encountered.
